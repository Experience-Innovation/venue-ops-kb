# Incident Command System (ICS) for Venues

Standardized command, control, and coordination framework that venues align to for interoperability with fire, EMS, and law enforcement during emergencies.

## Overview

The Incident Command System (ICS) is a scalable management structure with five functional areas: Command, Operations, Planning, Logistics, and Finance/Administration. Venues adopt ICS to ensure seamless interoperability with external emergency services during incidents.

## Structure

- **Command:** Incident Commander, Public Information Officer, Safety Officer, Liaison Officer
- **Operations:** Tactical execution of incident response
- **Planning:** Situation assessment, resource tracking, documentation
- **Logistics:** Resources, facilities, food, communications, medical support
- **Finance/Administration:** Cost tracking, procurement, compensation

## Key Standards

- ICS Canada provides the Pan-Canadian framework for standardized incident management
- FEMA ICS training (IS-100, IS-200, IS-700) establishes baseline competency
- NFPA 1026 (2024 Edition) defines competency standards for Incident Commanders
- Unified Command variant used for multi-agency, multi-day, or multi-stakeholder events

## Venue Application

Venues pre-designate an Incident Commander and establish a command post location. Pre-event briefings align venue staff, security, medical, and external responders on ICS structure. Tabletop exercises validate ICS activation protocols.
